# 长上下文的成本经济学

## 主题判断
1M 上下文正在成为产品标配，但“默认全塞上下文”并不等于最优解，规模化后成本会迅速放大。

## 关键动态
- Karpathy 指出长上下文请求成本近似线性增长，RAG 与向量检索在高并发场景仍可能更划算。
- Anthropic 的 1M 上下文发布说明能力边界被继续推高，但企业侧仍需精算单位查询成本。

## 实践启示
- 先算账再选架构：按查询频次、上下文长度、命中率比较“长上下文直塞”与“检索增强”。
- 用混合策略控制成本：高价值低频任务走长上下文，常规高频任务走 RAG。
