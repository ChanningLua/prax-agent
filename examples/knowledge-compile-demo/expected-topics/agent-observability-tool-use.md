# Agent 可观测性与工具调用能力

## 主题判断
Agent 从“能跑”进入“可治理”阶段：没有可观测性与精细化 tool-use 训练，成本和稳定性都会快速失控。

## 关键动态
- HN 案例显示：30 分钟任务因坏工具重复调用 200 次，静默消耗 14 美元，暴露缺少监控与告警的系统性风险。
- 讨论共识是必须追踪每个工具调用次数与成本，而不是只看最终输出质量。
- 中文社区从训练侧补充原因：高质量 agent 轨迹数据不足、RL 阶段 reward shaping 粗糙，会直接影响工具调用稳定性。

## 落地建议
- 最低配置：`per-tool call count`、成本上限、异常重试熔断、任务级告警。
- 训练与评估并重：把真实工具链轨迹纳入训练集和回归评测，而不只做离线问答评测。
